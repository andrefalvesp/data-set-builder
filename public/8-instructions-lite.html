INSTRUÇÕES PARA EXECUÇÃO DA ATIVIDADE E USO DA APLICAÇÃO

Esta atividade de construção de conjunto de dados consiste em elaborar

(1) enunciados de requisições (consultas) de informação que seja útil a um gestor de processos a partir de logs de eventos e

(2) as respectivas consultas em SQL equivalentes que permitam recuperar a informação requerida pelo enunciado em língua natural a partir do log de eventos.

Considere as seguintes restrições para a elaboração das requisições de informação:

  1. O log é a única fonte de dados disponível para a recuperação de informação.
  2. Apenas declarações em SQL puro devem ser elaboradas; ou seja, não é esperado o uso de cursores ou estruturas de fluxo de controle de linguagens disponibilizadas em SGBDs, como PgPLSQL, PLSQL, TransactSQL e similares.

Para guiar e facilitar a atividade, serão apresentados trechos de log sobre os quais vocês devem se basear para elaborar as consultas. Vocês devem considerar o contexto apresentado no trecho do log como a fonte de informação necessária para executar a consulta SQL; porém, a consulta SQL elaborada deve valer para o log completo, de onde o trecho foi retirado.

Abaixo seguem as instruções detalhadas de como vocês deverão proceder na execução da atividade. Por favor, leiam com atenção. Se, no decorrer da atividade, vocês tiverem dúvidas, acessem esse documento novamente. Um link para ele estará disponível em todas as telas da aplicação.

INSTRUÇÕES DETALHADAS:

  1. Para cada requisição a ser criada, a atividade a ser executada envolve a execução de três tarefas:
  1. Elaborar uma requisição de informação, expressa em língua natural (primeiramente em português), com base na análise do trecho de log que lhe é apresentado. Veja abaixo um exemplo genérico, o qual não considera o contexto de mineração de processos, retirado de Yu et al. (2019):

Qual é a média de expectativa de vida em países onde o inglês não é a língua oficial? 

  1. Traduzir a requisição de informação para inglês, como o exemplo abaixo:

What is the average life expectancy in the countries where English is not the official language?

  1. Elaborar a consulta em SQL equivalente que recupera a informação requisitada de acordo com a consulta em língua natural, como o exemplo abaixo:

SELECT AVG(life_expectancy)

FROM country

WHERE name NOT IN    

(SELECT T1.name    

FROM country AS T1 JOIN country_language AS T2 ON T1.code = T2.country_code    

WHERE T2.language = "English" AND T2.is_official = "T")

 

  1. Para cada trecho de log que lhe for apresentado, você deverá elaborar dois conjuntos de requisição em português + requisição em inglês + consulta em SQL.

  1. Para cada requisição a ser elaborada, três comandos SQL serão aleatoriamente apresentados a você. Você deverá escolher pelo menos um desses três comandos para incluir em sua consulta SQL. Os três comandos SQL apresentados aleatoriamente serão escolhidos a partir da seguinte lista:

SELECT com operações aritméticas

INNER JOIN

WHERE -- ALL

AVG / COUNT / MIN / MAX / SUM

GROUP BY

HAVING

ORDER BY

BETWEEN

LIKE

IS NULL / IS NOT NULL

UNION

INTERSECT

EXCEPT

IN / NOT IN

ANY / SOME

  1. O trecho de log apresentado como apoio deverá ser visto como uma relação (tabela) de um banco de dados relacional. Como esse banco de dados possui apenas uma tabela, para consultas envolvendo INNER JOIN, você deve considerar contextos em que os dados de uma tupla devem ser comparados com ela mesmo ou com as demais tuplas da própria tabela. Isso pode ser útil, por exemplo, para calcular diferenças de tempos de execução de atividades, buscar por atividades consecutivas no log, encontrar relacionamentos entre recursos de um mesmo caso, e comparar custo de atividades em casos diferentes.

  1. Você pode consultar o Google Tradutor para fazer traduções. Mas revise a tradução para garantir que a qualidade do enunciado da requisição em inglês esteja adequada.

  1. A sua consulta SQL precisa estar correta, ou seja, ao executarmos a sua consulta SQL sobre o log como uma relação (tabela), a consulta deve poder ser executada sem erros de sintaxe e obter um resultado que atende adequadamente o enunciado da requisição de informação em língua natural associada ao SQL.

  1. Interessam-nos consultas em SQL de dificuldade baixa, média, alta e super alta. Assim, no conjunto total de suas requisição em português + requisição em inglês + consulta em SQL, gostaríamos de encontrar pelo menos três níveis de dificuldade. Considere os exemplos abaixo como consultas de dificuldade baixa, média, alta e super alta.

  1. Dificuldade baixa (Yu et al., 2019):

SELECT COUNT(*)

FROM cars_data

WHERE cylinders > 4

  1. Dificuldade média (Yu et al., 2019):

SELECT T2.name, COUNT(*)

FROM concert AS T1 JOIN stadium AS T2 ON T1.stadium_id = T2.stadium_id

GROUP BY T1.stadium_id

  1. Dificuldade alta (Yu et al., 2019):

SELECT T1.country_name

FROM countries AS T1 JOIN continents AS T2 ON T1.continent = T2.cont_id JOIN car_makers AS T3 ON T1.country_id = T3.country

WHERE T2.continent = 'Europe'

GROUP BY T1.country_name

HAVING COUNT(*) >= 3

  1. Dificuldade super alta (Yu et al., 2019):

SELECT AVG(life_expectancy)

FROM country

WHERE name NOT IN     

(SELECT T1.name    

FROM country AS T1 JOIN country_language AS T2 ON T1.code = T2.country_code    

WHERE T2.language = "English" AND T2.is_official = "T")

SISTEMA DE PONTUAÇÃO:

  • Cada conjunto de requisição em português + requisição em inglês + consulta em SQL correta vale 1 ponto.
  • Para finalizar a atividade com excelência, você deve:
  1. Apresentar requisições úteis para o contexto de um gestor de processos.
  2. Apresentar requisições que exigem consultas SQL de diferentes níveis de dificuldade.
  3. Apresentar consultas SQL corretas.
  4. Alcançar uma pontuação de:
  • 4 pontos, se você for aluno da graduação;
  • 8 pontos, se você for aluno da pós-graduação.
  • Se você alcançou o número mínimo de pontos, mas pode fazer mais conjuntos de conjuntos de requisição em português + requisição em inglês + consulta em SQL, por favor, siga usando a aplicação. Para nós, quanto mais conteúdo melhor.

OBSERVAÇÕES:

  • Procure variar suas requisições em relação ao objetivo delas. Pense em: quem, o que, qual, quando, quanto, onde, por quanto tempo, etc. Além disso, lembre-se que um gestor de processos está interessando em diferentes objetivos, tais como: no fluxo de controle do processo, no relacionamento entre recursos, no desempenho de processos, no custo das atividades, no retrabalho, na associação de atividades e recursos, no tempo de trabalho, na checagem de conformidade, na melhoria de processos, em descrição, predição e prescrição, etc.
  • Você pode realizar essa atividade em diferentes dias e horários, conforme lhe convier. Não é necessário completar toda a atividade em uma única sessão de uso do sistema.
